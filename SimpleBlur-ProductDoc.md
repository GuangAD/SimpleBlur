## 1. 产品概述

SimpleBlur 是一款**纯前端、零服务器**的在线图片模糊工具。用户在浏览器本地完成全部图像处理，图片数据不会上传至任何服务器，保障隐私安全的同时无需注册、无水印、完全免费。

### 1.1 产品定位

| 维度 | 说明 |
|------|------|
| 目标用户 | 设计师、运营、内容创作者、普通网民等需要快速模糊图片的用户 |
| 核心价值 | 即开即用、无需安装；完全离线处理，隐私零泄露；支持3种专业模糊模式 |
| 竞品差异 | 相比同类工具：无水印、无上传、无广告、无注册；比客户端软件更轻量快速 |

### 1.2 核心特性一览

- 🚫 **无水印** — 输出图片与原图尺寸一致，无任何附加标记
- 🔒 **纯本地处理** — 全程 Canvas API，图片字节不离开浏览器
- 🌐 **双语界面** — 支持简体中文 / English，自动检测浏览器语言，手动可切换
- 📦 **单文件交付** — 仅 `index.html`，无依赖，可本地打开或静态托管
- 📱 **响应式布局** — 桌面/移动端自适应，支持触控框选
- ⚡ **实时预览** — 拖动滑块即时渲染，无需等待确认

## 2. 核心功能规格

### 2.1 图片上传

支持以下三种方式载入图片：

1. **点击上传** — 点击上传区域，弹出系统文件选择器
2. **拖拽上传** — 拖拽图片文件到上传区域（Drag & Drop）
3. **触摸上传** — 移动端触摸上传区域，调起相册或相机

**支持格式：** PNG · JPG/JPEG · WebP · GIF（取第一帧）

> 显示分辨率上限：800 px（长边），超出时等比缩小预览；**下载时恢复原始分辨率**。

---

### 2.2 区域选择（可选）

用户可在画布上拖拽绘制矩形选区，对**指定区域**进行精准模糊；若不绘制选区，则自动对**整张图片**应用模糊效果。

| 交互方式 | 行为 |
|----------|------|
| 鼠标拖拽 | 按下 → 移动 → 释放，绘制蓝色虚线选框 |
| 触控滑动 | `touchstart/move/end` 事件转发，与鼠标行为一致 |
| 不绘制选区 | 整张图片作为默认模糊目标 |
| 切换模式 | 自动清除当前选区并还原原图，避免效果叠加混乱 |

---

### 2.3 模糊模式

提供三种专业级模糊效果，均通过纯 JavaScript 在 OffscreenCanvas 上实现，**无第三方库依赖**：

#### Normal（普通 / 高斯模糊）

- **算法：** 三重 Box Blur 逼近高斯模糊（水平 + 垂直双通道，迭代 3 次）
- **效果：** 均匀柔化，适合隐私遮挡、背景虚化
- **参数映射：** 强度 0–100 → 模糊半径 0–40 px

#### Motion（运动模糊）

- **算法：** 多采样线性插值（最多 48 个采样点）
- **效果：** 带方向感的拖影，模拟高速运动或创意视觉效果
- **附加参数：** 方向角度 0°–360°（附实时方向指示盘可视化）

#### Radial（径向 / 缩放模糊）

- **算法：** 缩放采样叠加（最多 32 个采样点）
- **效果：** 从中心点向外扩散，模拟镜头缩放或旋涡效果
- **附加交互：** 点击画布设置径向中心点，蓝色光点实时标记位置；默认中心为图片/选区中心

---

### 2.4 模糊强度滑块

- **范围：** 0 ~ 100（默认值 **0**，方便用户对比观察变化）
- **实时预览：** 拖动滑块立即更新画布效果，无需点击"应用"
- **数值显示：** 滑块右侧实时展示当前数值（蓝色高亮）

---

### 2.5 操作按钮

| 按钮 | 行为与状态说明 |
|------|--------------|
| **Apply Blur（应用模糊）** | 将当前预览状态固化为新原图；清除选区；支持叠加多次模糊 |
| **Download Image（下载图片）** | 以原始分辨率导出 PNG；**上传图片后即可点击**，无需先应用模糊 |
| **Reset / New Image（重置）** | 清空画布、还原全部状态，准备处理新图片；图片消失后自动禁用 |

> Toast 通知：每次操作完成后，屏幕底部居中弹出文字提示（2.2 秒后自动消失，带弹入动效）。

## 3. 多语言支持

SimpleBlur 内置完整的 i18n 国际化系统，支持 **简体中文** 与 **English** 双语界面。

### 3.1 自动语言检测

页面加载时通过 `navigator.language` 检测浏览器首选语言：

- 浏览器语言以 `zh` 开头 → 自动切换为简体中文
- 其余语言 → 默认显示**英文**
- 用户手动切换语言后，偏好存入 `localStorage`，下次访问自动沿用

### 3.2 手动切换

页面**右上角**提供 `EN` / `中文` 双按钮切换器，当前语言高亮显示（蓝色背景）。切换即时生效，无需刷新页面。

## 4. 用户操作流程

### 4.1 典型路径 — 整图模糊

```
① 打开页面
   ↓
② 拖入 / 点击上传图片
   ↓
③ 右侧选择模糊模式（默认 Normal）
   ↓
④ 拖动强度滑块，实时预览效果
   ↓
⑤ 点击 Apply Blur → 效果固化（可继续叠加）
   ↓
⑥ 点击 Download Image → 原始分辨率 PNG 下载至本地
```

### 4.2 典型路径 — 局部模糊

```
① 上传图片
   ↓
② 在画布上拖拽绘制矩形选框（仅选区内预览模糊）
   ↓
③ 选择模糊模式与强度
   ↓
④ 点击 Apply Blur 固化该区域效果
   ↓
⑤ 可重新拖拽选框继续处理其他区域（重复 ②–④）
   ↓
⑥ 满意后下载
```

### 4.3 步骤指示器

右侧控制面板顶部提供 **4步骤指示器**（Step 1–4），随操作进度自动更新状态：

```
待完成（灰色圆圈） → 进行中（蓝色高亮） → 已完成（绿色对勾）
```

### 5.2 模糊算法详解

所有模糊算法均在独立的 **OffscreenCanvas** 上运算，处理完成后通过 `putImageData` 回写至预览 Canvas，保证原始图像数据在未点击"应用"前始终可恢复。

#### Normal Blur — 三重 Box Blur 近似高斯

```
水平通道 boxBlurH(data, tmp, w, h, r)
    ↕  ×3 次迭代
垂直通道 boxBlurV(tmp, data, w, h, r)

参数：radius = round(amount / 100 × 40)
时间复杂度：O(W × H × r)
```

与 CSS `filter: blur()` 效果近似，但可操作 `ImageData`，支持局部区域渲染。

#### Motion Blur — 方向采样插值

```javascript
// 核心逻辑伪代码
for each pixel (px, py):
  for s in 0..samples:
    t = s/(samples-1) - 0.5  // -0.5 ~ 0.5
    sx = px + t × halfLen × cos(angle)
    sy = py + t × halfLen × sin(angle)
    accumulate(srcData[sx, sy])
  out[px, py] = average
```

- 最多 **48 个采样点**（`amount=100` 时）
- 边界像素自动截断，不产生伪影

#### Radial Blur — 缩放采样叠加

```javascript
// 核心逻辑伪代码
for each pixel (px, py):
  for s in 0..samples:
    scale = 1 + (s/samples-1) × strength   // 1.0 ~ 1+strength
    sx = cx + (px - cx) / scale
    sy = cy + (py - cy) / scale
    accumulate(srcData[clamp(sx), clamp(sy)])
  out[px, py] = average

strength_max = 0.18  // 最大 18% 缩放，避免过度失真
```

### 5.3 分辨率处理策略

```
上传图片
  │
  ├─ 长边 > 800px ──→ 等比缩小至 800px（预览 Canvas）
  │                    ↓ 实时预览，流畅响应
  │
  └─ 长边 ≤ 800px ──→ 原尺寸绘制至预览 Canvas

点击下载
  │
  └──→ 创建全尺寸 Canvas（srcImg.naturalWidth × naturalHeight）
        ↓ drawImage(previewCanvas, 0, 0, fullW, fullH)
        ↓ toDataURL('image/png')
        ↓ 输出原始分辨率 PNG
```

---

### 5.4 状态管理

```javascript
// 核心状态变量
let srcImg        // 原始 Image 对象
let originalData  // ImageData 快照（未应用模糊时的原图）
let displayScale  // canvas像素 / 图片像素 缩放比
let selection     // {x, y, w, h} 选区坐标（canvas像素）
let mode          // 'normal' | 'motion' | 'radial'
let blurAmount    // 0–100
let motionAngle   // 0–360°
let radialCenter  // {x, y} 径向中心点
```

**关键设计：**
- `originalData` 每次点击 Apply 后更新为当前 Canvas 状态，实现**叠加模糊**
- 切换模式时执行 `ctx.putImageData(originalData, 0, 0)` 还原原图，避免效果残留

## 7. 部署与集成

### 7.1 部署方式

SimpleBlur 是**单文件静态应用**，支持以下任意部署方式：

1. **本地打开** — 直接双击 `index.html`，无需服务器
2. **静态托管** — 上传至 Vercel / Netlify / GitHub Pages / 腾讯云 COS 等
3. **内网部署** — 嵌入企业内网静态资源服务器，供内部员工使用

### 7.2 依赖与兼容性

| 项目 | 说明 |
|------|------|
| 外部依赖 | **无**（零 CDN、零 npm 依赖） |

## 8. 已知限制与后续规划

### 8.1 当前版本限制

- 大尺寸图片（> 4000px）在低端设备上处理耗时较长，暂无 Web Worker 卸载
- Motion / Radial 模式为纯 JS 软件渲染，处理速度低于 GPU 加速方案
- GIF 仅处理第一帧，不支持动图输出
- 不支持多步撤销（仅保留"当前原图"快照，Apply 后不可反悔）

### 8.2 后续迭代方向（Roadmap）

| 优先级 | 功能 | 说明 |
|--------|------|------|
| P0 | Web Worker 异步处理 | 将模糊计算移至 Worker，避免 UI 卡顿 |
| P0 | 多步撤销（Undo） | 维护历史快照栈，支持 Ctrl+Z 撤销 |
| P1 | 多区域叠加管理 | 显示已应用模糊区域列表，支持单独删除 |
| P1 | 椭圆 / 自由形选区 | 扩展选区类型，支持人脸区域更精准遮挡 |
| P1 | WebGL 加速 | GPU Shader 实现高质量大图模糊，大幅提速 |
| P2 | 更多语言支持 | 日语、韩语、德语、法语等 |
| P2 | 批量处理 | 支持一次上传多张图片，统一应用参数导出 |
